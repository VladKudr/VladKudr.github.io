Самостоятельная установка OpenClaw

Арендуем VPS-сервер, устанавливаем OpenClaw, подключаем Kimi K2.5 как модель. Работаем с Mac — никаких GUI на сервере не нужно.

~30 минут
macOS + VPS
Ubuntu 22.04

1

Что понадобится

Перед стартом убедись, что у тебя есть:

Mac с Terminal — стандартный терминал (или iTerm2)
Аккаунт на RU-VDS — ruvds.com, карта для оплаты
Аккаунт на платформе Kimi — platform.moonshot.ai, нужен API-ключ
~30 минут времени и стабильный интернет
ℹ️
Почему именно Kimi K2.5?

Kimi K2.5 через NVIDIA Moonshot API позволяет использовать мощную модель на бесплатной квоте. Это один из самых дешёвых способов запустить умного агента без ежемесячной подписки.

2

Выбор и аренда VPS на RU-VDS

Заходим на ruvds.com, регистрируемся (или входим) и создаём новый сервер.

Рекомендуемая конфигурация для OpenClaw:

ПараметрЗначение
ОСUbuntu 22.04 LTS Рекомендую
CPU2 vCPU
RAM4 GB
Диск40 GB SSD
ТрафикНеограниченный или 200+ GB
Цена~1 100 ₽/месяц (тариф «Турбо»)
1

Открой раздел «Заказать VDS»

В личном кабинете RU-VDS нажми «Создать сервер» или «Заказать VDS».

2

Выбери ОС Ubuntu 22.04 LTS

В списке образов выбери Ubuntu 22.04 LTS. Не бери 24.04 если хочешь максимально проверенный путь — большинство инструкций написано под 22.04.

3

Добавь SSH-ключ (сделаем на следующем шаге)

На этапе создания сервера будет поле «SSH-ключи». Пока пропусти его — на следующем шаге мы сгенерируем ключ и добавим его в личный кабинет, а сервер можно создать с паролем, добавив ключ позже.

4

Сохрани IP-адрес и root-пароль

После создания сервера RU-VDS покажет IP-адрес и временный пароль root. Запиши их — они понадобятся для первого входа.

⚠️
Сохрани пароль сразу

RU-VDS показывает root-пароль только один раз при создании. Если потеряешь — придётся делать сброс через панель управления.

3

SSH-ключи: генерация и добавление

SSH-ключ — это как замок и ключ: приватный ключ хранится у тебя на Mac, публичный — на сервере. Вход без пароля, быстрее и безопаснее.

1

Открой Terminal на Mac

Нажми Cmd + Пробел, введи «Terminal», Enter.

2

Сгенерируй ключ

Выполни команду:

terminal — mac 
ssh-keygen -t ed25519 -C "ruvds-openclaw"

Когда спросит путь — нажми Enter (оставить по умолчанию). Когда спросит пароль — тоже можно нажать Enter дважды (без пароля, для удобства).

В результате создадутся два файла:
— ~/.ssh/id_ed25519 — приватный ключ (никому не показывай)
— ~/.ssh/id_ed25519.pub — публичный ключ (его загружаем на сервер)

3

Скопируй публичный ключ

terminal — mac 
cat ~/.ssh/id_ed25519.pub

Скопируй всю строку — она начинается с ssh-ed25519 и заканчивается на ruvds-openclaw.

4

Добавь ключ в RU-VDS

В личном кабинете RU-VDS найди раздел «SSH-ключи» (обычно в настройках профиля или в боковом меню). Нажми «Добавить ключ», вставь скопированную строку, дай имя — например MacBook, и сохрани.

Теперь при создании следующего сервера можно будет сразу привязывать этот ключ.

4

Подключение к серверу с Mac

Берём IP-адрес сервера из панели RU-VDS и подключаемся через Terminal.

1

Подключись по SSH

terminal — mac 
ssh root@ВАШ_IP_АДРЕС

Замени ВАШ_IP_АДРЕС на реальный IP из панели RU-VDS. При первом подключении терминал спросит подтверждение — введи yes и нажми Enter.

Если подключаешься с паролем (без ключа) — введи root-пароль, который сохранил при создании сервера.

2

Обнови систему

Первое, что делаем на свежем сервере:

ssh — сервер 
apt update && apt upgrade -y

Подождём 1–2 минуты, пока обновятся пакеты.

💡
Упрости повторные входы

Добавь алиас в ~/.ssh/config на своём Mac — тогда можно будет писать просто ssh openclaw вместо полного IP каждый раз. Открой файл: nano ~/.ssh/config и добавь блок:
Host openclaw
  HostName ВАШ_IP
  User root
  IdentityFile ~/.ssh/id_ed25519

5

Установка OpenClaw на сервер

Находясь в SSH-сессии на сервере, запускаем официальный установочный скрипт. Он автоматически определит ОС, установит Node.js и сам OpenClaw.

1

Запусти установщик

ssh — сервер 
curl -fsSL https://openclaw.ai/install.sh | bash

Скрипт скачает и установит Node.js 22+, затем установит OpenClaw и запустит интерактивный мастер настройки (onboarding).

2

Пройди мастер настройки

После установки появится текстовый интерфейс. Отвечай так:

ВопросТвой ответ
Disclaimer / рискиYes → Enter
Onboarding modeQuickStart
AI providerПропусти (выберем Kimi вручную на шаге 7)
Gateway portEnter (оставить 18789)
Gateway bindEnter (127.0.0.1 — loopback)
Gateway auth tokenEnter (генерируется автоматически) — сохрани токен!
Chat channel (Telegram и т.д.)Skip for now
TailscaleNo / Enter
3

Сохрани Gateway Token

В конце установки OpenClaw покажет Gateway Token — длинную строку. Это твой пароль для входа в веб-интерфейс. Скопируй его и сохрани в надёжное место (заметки, менеджер паролей).

4

Проверь статус

ssh — сервер 
openclaw --version
openclaw gateway status

Должны увидеть версию OpenClaw и статус running у Gateway.

⚠️
Если команда openclaw не найдена

Перезагрузи сессию: source ~/.bashrc или выйди и зайди снова по SSH. PATH обновится и команда заработает.

6

Получение API-ключа Kimi K2.5

Открой в браузере на Mac платформу Kimi и получи ключ для подключения к OpenClaw.

1

Зарегистрируйся на Moonshot AI

Перейди на platform.moonshot.ai и создай аккаунт. После регистрации пополни баланс — новым аккаунтам нужно минимальное пополнение для выхода из бесплатного тира.

2

Создай API-ключ

В личном кабинете перейди в раздел API Keys → Create API Key. Дай ключу имя (например openclaw-server) и скопируй его.

3

Выбери модель kimi-k2.5

В настройках или в документации убедись, что ключ имеет доступ к модели moonshot/kimi-k2.5 — это основная модель, которую будем использовать в OpenClaw.

💡
Альтернатива: NVIDIA NIM (бесплатно)

Kimi K2.5 также доступна через NVIDIA Moonshot API на build.nvidia.com — там есть бесплатная квота. Ключ начинается с nvapi-.... Настройка аналогична, только base_url будет другим.

7

Подключение Kimi в настройках OpenClaw

Теперь добавим Kimi K2.5 как провайдера прямо через конфиг на сервере.

1

Открой файл конфигурации

ssh — сервер 
nano ~/.openclaw/openclaw.json
2

Добавь провайдера Kimi

Найди секцию "model" или добавь её, если нет. Пример итоговой конфигурации:

~/.openclaw/openclaw.json 
{
  "model": {
    "provider": "moonshot",
    "name": "kimi-k2.5",
    "apiKey": "ТВОЙ_API_КЛЮЧ_KIMI"
  }
}

Замени ТВОЙ_API_КЛЮЧ_KIMI на реальный ключ из шага 6. Если в файле уже есть другие настройки — добавляй секцию model внутрь существующего JSON, не заменяй весь файл.

3

Сохрани и перезапусти Gateway

В nano: Ctrl+O → Enter → Ctrl+X. Затем:

ssh — сервер 
openclaw gateway restart
4

Проверь что всё работает

ssh — сервер 
openclaw doctor
openclaw gateway status

Команда doctor проверит конфигурацию и покажет, если что-то настроено неверно.

8

Доступ к интерфейсу с Mac через SSH-туннель

OpenClaw работает на сервере на порту 18789, но снаружи не доступен (это правильно с точки зрения безопасности). Чтобы открыть веб-интерфейс на своём Mac, используем SSH-туннель.

1

Открой SSH-туннель с Mac

В новой вкладке Terminal на Mac (не на сервере) выполни:

terminal — mac (новая вкладка) 
ssh -L 18789:localhost:18789 root@ВАШ_IP_АДРЕС -N

Флаг -N означает «только туннель, без входа в шелл». Оставь эту вкладку открытой — туннель работает пока она живёт.

2

Открой веб-интерфейс в браузере

Перейди в браузере на Mac по адресу:

http://127.0.0.1:18789

Введи Gateway Token, который сохранил на шаге 5. Ты попадёшь в интерфейс OpenClaw.

3

Проверь подключение к Kimi

В интерфейсе перейди в Overview — там должен отображаться статус Connected. Затем открой вкладку Chat и отправь первое тестовое сообщение.

💡
Удобнее: постоянный туннель в фоне

Чтобы не держать отдельную вкладку, запускай туннель в фоне:
ssh -L 18789:localhost:18789 root@ВАШ_IP -fNT
Флаг -f уходит в фон, -T отключает псевдотерминал. Закрыть его можно через pkill -f "18789:localhost".

✅
Готово!

OpenClaw запущен на твоём VPS, подключён к Kimi K2.5, и доступен с Mac через безопасный SSH-туннель. Сервер работает 24/7 — агент всегда онлайн, даже когда Mac закрыт.
