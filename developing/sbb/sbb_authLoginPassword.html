Аутентификация по паре логин/пароль

Что планируем сделать

В задаче аутентификации по паре логин/пароль самой трудной частью является расчет свертки пароля, поэтому мы уделим ей наиболее пристальное внимание, но пройдем по также по всему процессу получения токена сессии.

Вызов метода preLogin

Вы выполняете первым шагом вызов preLogin со структурой аналогичной ниже:

SOAP запрос preLogin XML SOAP запрос из Java SOAP запрос SoapUI

XML структура soap запроса preLogin
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:upg="http://upg.sbns.bssys.com/">
   <soapenv:Header/>
   <soapenv:Body>
      <upg:preLogin>
         <upg:userLogin>?</upg:userLogin>
         <!--Optional:-->
         <upg:changePassword>?</upg:changePassword>

      </upg:preLogin>
   </soapenv:Body>
</soapenv:Envelope>
Пример вызова preLogin из Java
URL url;
UniversalPaymentGate universalPaymentGate = new UniversalPaymentGate(url);
List<byte[]> preLoginResponse = paymentGate.getUniversalPaymentGateSbrfImplPort().preLogin(userLogin, false);
